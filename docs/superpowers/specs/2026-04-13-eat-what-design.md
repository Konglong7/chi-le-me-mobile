# 吃了么 Mobile Web / Android 设计说明

## 目标

基于 `eat what 原型图.html` 还原一个手机端优先的产品界面，提供昵称进入、提案、投票、组队、聊天、转盘、分享、历史、设置 8 个核心页面，并保留 Android Studio 可运行的 Android 包装版本。

## 范围

- 先实现移动端 Web 单页应用
- Android 版本复用同一套前端页面
- 数据先使用本地持久化，保证刷新或重新打开后仍可恢复
- UI 以现有原型为基准，颜色、布局、层级、交互尽量一比一还原

## 交互结构

1. 昵称进入页
2. 首页
3. 发起提案页
4. 提案详情页
5. 幸运转盘页
6. 分享美食页
7. 历史记录页
8. 设置页

## 架构

- 前端：`Vite + React + TypeScript + Tailwind CSS`
- 状态：React Context + Reducer
- 持久化：`localStorage`
- Android：`Capacitor`

## 设计原则

- 手机端优先，主内容宽度控制在 430px 内
- 首页、转盘、历史保留底部导航
- 关键视觉元素保持原型中的红色主题、圆角、阴影、渐变与层级
- 允许在实现时将静态原型调整成真实可交互页面，但不改变视觉方向

## 非目标

- 当前版本不接入远程后端
- 当前版本不做真实多用户同步
- 当前版本不做图片上传到服务器
