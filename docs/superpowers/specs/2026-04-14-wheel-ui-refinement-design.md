# 吃了么转盘页 UI 精修设计说明

## 目标

在保持转盘页深色沉浸式氛围的前提下，重点解决当前幸运转盘的两个问题：

- 奖项文字位置与朝向不准确，和参考转盘相比缺少“贴合扇区”的稳定感
- 底部导航在深色页面中的字体、图标、间距与整页视觉不够统一

本设计只覆盖 `src/screens/wheel-screen.tsx` 与 `src/components/layout.tsx` 所涉及的转盘页 UI 精修，不改变当前转盘抽取逻辑、数据来源和页面信息架构。

## 直接采用的默认假设

基于本轮确认，直接采用以下假设：

1. 转盘页继续保留深色沉浸式背景，不切回浅色导航方案
2. 文字布局以用户提供的 HTML 原型和参考站点 `https://tinhour.github.io/spin-the-wheel/` 的扇区内文字摆放方式为主要参考
3. 现有 React 状态、随机选中逻辑、动画时长与结果展示机制保持不变
4. 本次工作目标是“视觉与排版精修”，不是重做整套转盘功能

## 当前问题

### 文字层

当前实现把每个奖项文字放在统一半径的圆环上，再通过反向旋转把文字扶正。这个方案有三个直接问题：

- 所有文字共享同一条半径轨道，视觉上像悬浮在盘面上，而不是嵌在扇区里
- 长中文名和短英文名使用同一套宽度与字号，中心位置容易不均衡
- 当奖项数量变化时，文字与扇区边界的关系不稳定，容易显得偏里或偏外

### 底部导航

当前深色底部导航可以使用，但仍存在统一性问题：

- 图标与文字的字重、明度、上下间距不够协调
- 激活态和非激活态的层级差距偏弱
- 导航容器的模糊、描边、阴影和转盘页上方玻璃卡片不是同一套视觉语言

## 方案比较

### 方案 A：继续沿用圆周文字布局，只微调参数

做法：

- 保留当前 `rotate -> translateY -> reverse rotate` 模型
- 手工微调文字半径、宽度、字号和行高

优点：

- 改动最小
- 几乎不影响现有结构

缺点：

- 只能缓解，不能真正解决“文字不在扇区里”的问题
- 中文长词和英文短词的视觉中心仍不稳定
- 和参考转盘的质感差距仍然明显

### 方案 B：改为扇区容器内文字布局

做法：

- 每个奖项拥有一个独立的扇区文字容器
- 容器先旋转到该奖项扇区中心角
- 文字块在容器内部沿扇区法线方向摆放，再根据长度做宽度和两行压缩

优点：

- 最接近参考站点的布局逻辑
- 文字和扇区绑定后，位置更稳定、更像真实轮盘
- 仍可继续沿用现在的 CSS 渐变盘面与旋转动画

缺点：

- 需要改造当前标签层结构
- 需要补一层布局计算，复杂度高于纯参数微调

### 方案 C：改成 SVG 轮盘

做法：

- 用 SVG 统一绘制扇区、文字、外圈和高光

优点：

- 精度最高
- 后续扩展复杂效果更灵活

缺点：

- 对当前任务而言改动过大
- 会引入额外实现和测试成本

## 推荐方案

采用 `方案 B：扇区容器内文字布局`。

原因：

- 它能直接解决本轮最核心的问题，即“字体位置要正确摆放”
- 它和现有 React 页面、渐变扇区、随机旋转逻辑兼容，不需要推翻当前实现
- 它能在有限改动范围内，把视觉效果拉近到用户给出的 HTML 原型与参考轮盘

## 设计方案

### 1. 转盘结构

转盘仍由三层组成：

1. 外层：深色背景中的轮盘容器、指针、中心按钮
2. 中层：旋转的扇区盘面，继续使用现有转盘旋转动画
3. 内层：随盘面一起旋转的文字层与高光层

结构调整原则：

- 结果判定逻辑继续基于当前旋转角度计算
- 视觉层可以重构，但不改变 `wheelRotation` 与 `animatedRotation` 的语义
- 中心按钮继续放在盘面中央，由透明点击层触发

### 2. 奖项文字布局

每个奖项改为“扇区文字节点”而不是“圆周标签”。

布局规则：

- 每个节点的基准角度为当前扇区中心角
- 节点容器以转盘中心为原点旋转到目标角度
- 文字块放在容器内部的固定径向距离上，而不是直接挂在圆周边缘
- 文字块默认沿扇区切线方向摆放，即相对扇区半径旋转 `90deg`，不再统一保持绝对水平
- 中文长名称在必要时允许换成两行，短名称保持单行
- 文字最大宽度、字号、字重与行高按奖项数量动态收紧

视觉目标：

- 文字位于盘面半径约 `55% - 68%` 的范围内，落在扇区视觉中心偏外区域
- 不贴近外圈装饰边，也不压到中心按钮
- 在 6 项默认场景下呈现与原型相近的“顺着扇区走”的摆放效果

### 3. 奖项文字适配规则

为避免不同名称长度导致布局失衡，增加轻量规则化处理：

- `1-4` 个中文字符或短英文词：单行展示
- 较长中文名称：允许拆成两行，每行保持居中
- 超长内容不追求完整铺满扇区，以视觉稳定优先，必要时缩小字号
- 空状态占位文案仍然支持展示，但采用更克制的字号和透明度

本次不引入滚动、跑马灯或路径文字。

### 4. 转盘视觉层级

保留当前页面的深色沉浸式基调，但把轮盘本体做得更完整：

- 外圈描边、高光和阴影要明显区分“盘体”和“背景”
- 中心按钮采用更稳定的尺寸、边框和投影，避免看起来过轻
- 指针与盘面顶部的距离、尺寸和阴影重新收敛，确保视觉焦点先落在指针和中心按钮之间
- 文字层与盘面扇区颜色保持足够对比，避免在亮色扇区上发白发虚

### 5. 深色底部导航

转盘页继续使用深色导航，但统一成一套更稳定的版式。

导航规则：

- 导航容器使用半透明深色面板，带轻微模糊和上边界分隔
- 图标与文字统一对齐到同一视觉中心，避免图标偏上、文字偏下
- 标签字号控制在 `11px` 左右，字重采用 `medium` 级别
- 激活态使用主题红并增强明度；未激活态保持低对比灰白
- 保留三项导航，不改变信息架构与点击逻辑

统一目标：

- 转盘页导航看起来是这页视觉系统的一部分，而不是浅色页面导航简单换色
- 导航的间距、字重、透明度与上方玻璃卡片风格一致

## 组件边界

### `src/screens/wheel-screen.tsx`

负责：

- 计算扇区数据
- 输出新的奖项文字层结构
- 保持现有旋转动画和转盘交互

不负责：

- 抽奖逻辑改写
- 全局导航规则定义

### `src/components/layout.tsx`

负责：

- 调整 `dark` 变体底部导航的字体、图标、间距和容器样式
- 保持浅色页面导航外观不回归

不负责：

- 转盘内部结构

### `src/styles.css`

负责：

- 补充转盘文字布局、导航层级和必要动画细节的通用样式

不负责：

- 页面级业务逻辑

## 测试与验收

### 视觉验收

- 6 个默认选项时，奖项文字位于各自扇区中心区域，视觉上不飘
- 长中文名和短英文名在盘面上都不会明显错位
- 转盘旋转过程中，文字跟随盘面转动，不出现抖动或层级穿插
- 深色底部导航在转盘页中与整体视觉一致，字体和图标比例协调

### 行为验收

- 点击中心按钮仍可正常触发抽奖
- 转盘停止后结果文案与指针指向保持一致
- 空选项、少量选项和默认 6 项场景都能正常渲染
- 底部导航切页行为不变

### 回归范围

- `wheel-screen` 的交互测试需要继续通过
- 不应影响首页、历史页等使用相同底部导航组件的页面结构

## 非目标

- 本次不重做整页色板
- 本次不新增音效、粒子、领奖弹窗等互动效果
- 本次不改造转盘数据模型
- 本次不把转盘改成 SVG 或 Canvas

## 实施顺序

1. 重构轮盘文字层数据结构与节点布局
2. 调整轮盘容器、中心按钮、指针和高光层级
3. 统一深色底部导航的字体、图标和间距
4. 跑现有测试并补充必要回归

## 最终结论

本次 UI 精修将以“扇区容器内文字布局”为核心改造点，在不动抽奖逻辑的前提下，解决字体位置不准的问题，并同步统一深色底部导航的字体和版式，使幸运转盘页更接近原型和参考轮盘的完成度。
